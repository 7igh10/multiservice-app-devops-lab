О проекте

Demo frontend для проверки контейнера `frontend`, конфигурации `default.conf` и корректной раздачи статических файлов без сложного приложения на JavaScript.

Маршрутизация

Если вы открыли эту страницу по домену или IP, значит фронтенд-сервис доступен, а `nginx` успешно обслуживает стартовую точку приложения.

Наблюдаемость

Для production сюда обычно добавляют health-check, access log, error log, rate limiting и метрики по проксируемым запросам.

Назначение

Это тестовая страница, но не пустой шаблон. Она нужна, чтобы быстро увидеть, что деплой действительно живой и визуально, и технически.

Что проверить после выката

  1. Открывается ли `index.html` без `404` и `502`.
  2. Правильно ли работает проксирование запросов к Python backend.
  3. Отдаются ли статические файлы с ожидаемыми заголовками.
  4. Нет ли ошибок в браузерной консоли и сетевых таймаутов.
  5. Совпадает ли поведение с blue-green сценарием деплоя.

Frontend + Python backend

Сценарий проверки

Обычно этот frontend кладут в контейнер `nginx`, а backend поднимают отдельно на Python. Если API отдает `200 OK` на health endpoint, индикатор выше переключится в зеленый статус. Если endpoint не настроен, страница останется рабочей и просто покажет предупреждение.

Этапы smoke-test

  • Проверить доступность главной страницы.
  • Убедиться, что статика не кешируется ошибочно при релизе.
  • Проверить backend health route через frontend proxy.
  • Посмотреть nginx access/error logs при первом запросе.
